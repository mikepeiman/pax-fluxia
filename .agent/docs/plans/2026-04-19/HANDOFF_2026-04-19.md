### `9329bab7` ÔÇö MG-PERF Phase A

**Files:**
- `src/lib/territory/families/metaballGrid/MetaballGridFamily.ts` (main changes)
- `src/lib/territory/families/metaballGrid/metaballGridStats.ts` (new Svelte writable store)

- `src/lib/config/game.config.ts` (three new tunables)

**What it does:**

1. **Dirty-flag paint gate** ÔÇö `MetaballGridFamily.update()` now computes a `paintSig` from every input that can change the painted output. When the signature matches the previous frame's, the existing `PIXI.Graphics` draw list remains visible at zero cost ÔÇö no scene build, no paint. Progress is quantized to 1/2048 steps so micro-changed progress values don't thrash the gate.
2. **Plan-params cache invalidation** ÔÇö `lastPlanParamsKey` field fingerprints spacing/origin/distribution/jitter/maxCells. Previously the plan only rebuilt on `transitionKey` change, which hid settings-UI edits until the next conquest. Now edits to these five knobs propagate immediately.
3. **Three new tunables:** `METABALL_GRID_DISTRIBUTION` (`square | hex_offset | jittered`), `METABALL_GRID_POSITION_JITTER` (0..0.5 fraction of spacing), `METABALL_GRID_MAX_CELLS` (hard cap; 0 = uncapped).
4. **Live stats store** ÔÇö `metaballGridStats` Svelte writable emits `{ requestedSpacingPx, effectiveSpacingPx, totalCells, emittableCells, paintedCells, lastUpdateMs, emaUpdateMs, lastFrameSkipped, frameCount, skippedFrameCount }` each update. Consumed by the new "Perf" module in `MetaballGridTuning.svelte` for live readouts.

**Known limitation introduced in Phase A:** the initial paint-sig construction did **not** include any classification/geometry identity fingerprint. This was the latent cause of bug #2 below, fixed in `6b17b5e3`.
