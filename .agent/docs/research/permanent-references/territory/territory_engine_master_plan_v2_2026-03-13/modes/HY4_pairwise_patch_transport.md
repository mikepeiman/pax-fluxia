# HY4 Pairwise + Patch + Transport

## Plain-Language Explanation

`HY4` uses exact pairwise arrangement-style static truth from `FG4` and combines it with patch and transport-oriented dynamic behavior. It is the most exactness-driven hybrid concept in the set.

## Current Implementation State

- Status: partial
- Native maturity: not yet native end-to-end
- Current role: selectable comparison hybrid plan

## Current Live Routing And Backend Reality

- Live only when `hybrid` mode is active and `HY4` is selected
- Current backend reality: adapter-backed through `PVV2`
- Current static leg: `FG4`
- Current dynamic leg: `DY4`

## Target Shape

`HY4` should become the native hybrid plan that applies exact arrangement truth first, then selectively layers patch or transport logic under strict gating rules.

## Backend Fit

- `PVV2`: useful current comparison surface for exactness-oriented validation
- `PVV3`: important long-term parity target
- `DF`: maintained parity target

## Dependencies

- `HY-000` common hybrid route contract
- `FG4` native arrangement publisher
- `DY4` native transport/correction path

## Atomic Tasks

- `HY-000` Define the common hybrid route contract and exclusivity semantics
- `HY4-001` Compose FG4 + DY2/DY4 gating and exactness rules

## Acceptance Checks

- Exact static arrangement truth remains authoritative
- Patch/transport overlays are gated and documented
- Multi-owner junction exactness survives dynamic refinement
