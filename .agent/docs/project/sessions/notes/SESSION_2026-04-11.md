## Rendering-ideas synthesis added this round

- The new rendering dumps are valuable, but the main value is **classification**, not direct mode proliferation.
- Many brainstormed "methods" are not actually top-level render families; they are better understood as choices along separate axes:
  - canonical truth model
  - family internal representation
  - transition representation
  - border derivation method
  - sampling / polygonization
  - presentation styling
- This axis-based taxonomy should become part of the future `IDEAS_LEDGER_RENDERING_AND_GAMEPLAY.md` so the project stops turning every promising internal technique into a new first-class runtime category.

### Metaball-specific takeaways

- `metaball` should remain a family-level representation and transition strategy, not the canonical source of ownership truth.
- Re-deriving borders from the current metaball scalar field each frame looks like a strong family-internal idea because it avoids fragile vector-to-vector border morphing.
- Fill-first / water-flow thinking is promising inside a transition window for `MetaballFamily`, provided bookend truth remains graph-native/canonical.
- Hex-aligned / triangular-lattice ideas belong under sampling and polygonization choices, not separate family identity.

### Practical ranking for metaball transition ideas

- Best near-term path:
  - per-node weight animation
  - then per-node + edge-tube influence
- Good experimental branch:
  - old/new field morph
- Research-only for now:
  - heuristic velocity field
  - level-set / OT / PDE-inspired flow

### Most valuable conceptual hybrid

- Canonical graph/frontier truth defines the start and end states.
- `MetaballFamily` may use scalar-field evolution during the transition window.
- Borders are re-derived per frame from the current field.
- The family snaps cleanly to canonical truth at transition end.
