## Runtime corrections after the UI split

- A more careful audit showed the earlier runtime-fix claim was overstated. The app and server are both using live `/common`, so the regressions were real behavior issues, not stale-package artifacts.
- Saved that investigation at `.agent/docs/project/implementation-plans/2026-04-12/MAPGEN_RUNTIME_REGRESSION_ANALYSIS_2026-04-12.md`.
- `Board Fill = 100%` now truly uses the full board in `common/src/mapgen/placement.ts`; direct probes now hit actual corner anchors instead of the old padded-interior extrema.
- `curved` lane mode in `common/src/mapgen/lanePolylines.ts` is now low-margin straight-first, but blocker-near chords can curve softly outward again so medium settings do not collapse into `all straight`.
- `LaneRenderer.ts` now trims nonlinear lane paths to star rims and strokes them as smooth curves to remove visible joints and endpoint overlap artifacts.
- Attack-surge heading now follows lane heading in `GameCanvas.svelte` instead of always using the direct attacker-to-defender chord.
- `ShipRenderer.ts` now always allows lane-polyline-based travel when a published lane path exists.
- `conquest/strategies.ts` now attaches published lane geometry for `travel` and `arrowhead` conquest modes when nonlinear lanes are available.
- Added a new territory toggle `TERRITORY_CX_CONTEST_MIDPOINT_VSTARS` so contested-lane midpoint pair virtual stars can be evaluated independently from main CX corridors.
- Runtime corridor injection now honors that split:
  - main CX can be off
  - contest-midpoint pairs can still be on
  - the midpoint feature no longer drags in broader distributed corridor sampling when used on its own
- `multiplayerStore.svelte.ts` now retries `seat reservation expired` failures when joining the lobby room.
- `GameRoom.ts` now uses a 120-second seat reservation timeout.
