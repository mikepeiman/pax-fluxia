# Session Notes - 2026-04-13

## Lane truth ownership

- User reported that lanes still disappear visually above a certain `Lane Margin` value while movement and attacks remain mechanically connected.
- User explicitly required that visible lane manifestation be derived only from map truth, not from a separate render-time interpretation layer.
- Tracing confirmed `/common` already carries lane-path truth through shared types and schema.
- The actual divergence was in SP/client ownership:
  - lane truth was being rebuilt into the cache
  - but not written back onto authoritative `state.connections`
  - `LaneRenderer` then applied its own star-gap visibility logic, creating a second lane interpretation layer

## Fixes applied

- `lanePolylines.ts`
  - removed reduced-clearance solving below requested Lane Margin
  - removed invalid straight fallback
  - lane-aware graph builder now preserves connectivity by selecting other satisfiable edges rather than forcing unsatisfied ones
- `mapgen/index.ts`
  - removed the final post-build lane rewrite
- `GameSettingsPanel.svelte`
  - lane-key config imports/presets now rebuild real lane geometry
- `gameStore.svelte.ts`
  - standard-map generation now writes lane truth onto authoritative connections
  - debug/rebuild paths now synchronize authoritative connections from the same rebuilt lane-aware source as the cache
  - live refresh paths now pass real edge distances into the shared lane-aware selector
  - saved-map export now preserves lane truth
  - saved-map import now reuses saved lane truth when present
- `lanePolylineCache.ts`
  - `rebuildLanePolylineCache()` now returns lane-aware connections
- `LaneRenderer.ts`
  - visible connection lanes now render directly from authoritative connection truth
  - removed renderer-side connection-lane shortening
  - removed separate intervening-star gap carving
  - canonicalizes bidirectional lane drawing so the same lane is not rendered twice
- `map.types.ts`
  - `MapDefinition.connections` now supports `laneWaypoints` and `lanePathKind`
- `game.config.ts`
  - reconciled missing arrow-config typings so client type-checking can pass again
  - added `MAPGEN_LANE_ADJUSTED_PATH_STYLE` as the unsurfaced backend control for `angular` vs `curved` remapped lanes

  - center of non-endpoint star to lane centerline
- Deterministic result from the frozen map:
  - false-positive curves were real and are now gone
  - above roughly `LM 175`, the strict all-pairs straight-only feasible graph is disconnected
