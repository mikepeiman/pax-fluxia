# Deep Document Processing Plan (Step 2)

## Goal

Read and process all project documentation to produce:
1. **Authoritative source-of-truth specs** — one definitive doc per topic, per scope
2. **A rewritten AGENT.md** — grounded in verified truths, updated paths, reasonable context length
3. **Clear, actionable task docs** — for completing the architecture refactor and geometry-renderer pipeline

## Methodology

Every document group is processed using:
- **Gold-Mining Heuristic** — 6 lenses (Treasure Hunt, Temporal Humility, Amnesia Awareness, Vision Integrity, Assumption Surfacing, Collaborative Intent)
- **Agent Council Personas** — 3-4 relevant personas per group
- **Output format** — 🏆 Gold Nuggets, 🔍 Assumptions, ❓ Questions for Discussion

## Processing Order (6 tranches, dependency-first)

1. **Territory Architecture & Geometry Pipeline** (~25 docs) → Consolidated TERRITORY_ARCHITECTURE v2
2. **VFX & Animation Systems** (~15 docs) → Consolidated VFX source-of-truth
3. **Game Design & Mechanics** (~12 docs) → Reconciled MECHANICS, updated PRD
4. **Engineering Architecture & Tech Stack** (~10 docs) → Current vs target architecture
5. **Planning Documents & Active Work** (~20 docs) → Clean active plan set, next-steps
6. **Agentic & Meta Docs → AGENT.md Rewrite** (~100+ docs) → Rewritten AGENT.md
