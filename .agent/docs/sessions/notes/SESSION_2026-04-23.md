### Architectural interpretation

- The async territory presentation queue is working and measurable. Stale territory frames are being superseded instead of committed serially.
- Direct order dispatch is now cheap, roughly `0.9ms` to `1.6ms`, which means the remaining gameplay latency is concentrated in the pointer-path UI flow and surrounding main-thread scheduling rather than the core order mutation itself.
- Metaball gameplay territory cost is now relatively light in steady state.
- Perimeter-Field load remains the strongest renderer-side spike, dominated by geometry build plus territory commit.
- Shared `MetaballRenderer` solve remains a strong common optimization surface for both Metaball-Grid and Perimeter-Field, but the current browser data also points to non-territory gameplay hotspots such as Pixi geometry packing, line batching, buffer upload, and ship rendering.
