# 1. The right design structure

Each player cosmetic should probably have **3 layers**, not one:

## A. Frontier layer

What happens near the boundary.

Examples:

* faint shimmer
* drifting sparks
* thin rune tracery
* dust/glow current moving along the border

## B. Interior ambient layer

What happens softly inside the region.

Examples:

* motes
* haze
* faint drifting streaks
* subtle symbol ghosts
* tiny flow lines

## C. Accent pulse layer

A rare intermittent effect.

Examples:

* slow heartbeat pulse from the star network
* faint energy wash
* occasional emblem apparition
* tiny wave / ripple through territory

That gives a premium feel.

---

# 2. Technical principle

These should be:

* **subtle**
* **low-frequency**
* **mostly shader-driven**
* **sparse particles only where necessary**
* **parameterized by player cosmetic profile**

Meaning:

A player cosmetic is mostly just a bundle of parameters:

```ts
type RegionFxProfile = {
  palette: Color[]
  borderGlowColor: Color
  ambientParticleType: "motes" | "embers" | "petals" | "glyphs"
  particleDensity: number
  flowStyle: "drift" | "swirl" | "wave" | "rising"
  interiorPattern: "noise" | "veins" | "cloud" | "runic"
  accentEvent: "pulse" | "ripple" | "flare" | "symbol"
  frontierBehavior: "sparkle" | "crawl" | "shimmer" | "arc"
}
```

That is much better than building every cosmetic as a totally separate effect.

# 5. Best technical implementation model

Do not make each cosmetic a unique one-off pile of particles.

Use a composable system:

# Region FX Stack

Each player gets up to 4 optional layers:

## Layer 1 — Interior shader

Cheap, always on.

Examples:

* nebula veil
* banner light
* shadow mist
* leyline flow

This should be a masked shader using:

* scrolling noise
* gradient map
* flow map
* distortion
* color palette

## Layer 2 — Sparse ambient particles

Optional, very low density.

Examples:

* stardust
* embers
* spores
* snow motes

This should be:

* pooled
* low count
* clipped to region bounds
* maybe simulated in local-space

## Layer 3 — Frontier treatment

Very strong identity for little cost.

Examples:

* border shimmer
* crawling sparks
* runic trace
* electric crawl

This is high value because borders are already visually important.

## Layer 4 — Accent event

Rare and beautiful.

Examples:

* a pulse every 8–15 seconds
* emblem apparition
* wave sweep
* glint surge

These make the cosmetic feel premium.

# 8. Performance-conscious rules

These must remain subtle and cheap.

## Rules

* no high-density particles everywhere
* no full-region blur passes per player if avoidable
* no many-sprite decal spam
* no random per-frame allocations
* no unique shader compilation per player

Instead:

* one shared shader
* player-specific uniforms
* one particle system template with swapped textures/colors
* one masked overlay per region class if possible

# 11. Best product architecture

If I were designing this for your game, I’d make cosmetics data-driven:

```ts
type CosmeticTheme = {
  id: string
  displayName: string

  interiorShaderPreset: string
  frontierShaderPreset: string
  particlePreset: string | null
  accentPreset: string | null

  palette: {
    primary: string
    secondary: string
    glow: string
    shadow: string
  }

  params: {
    intensity: number
    driftSpeed: number
    particleDensity: number
    accentFrequency: number
    borderBrightness: number
  }
}
```

That lets you build many cosmetics fast.
