### 1. Canonical Territory Pipeline — Compiler Stages

Built and wired the new canonical compiler pipeline end-to-end:
- **`metricStage.ts`** — Dijkstra ownership solve over the star graph; produces `MetricState` with `ownership[]` and `frontierEdges[]`
- **`frontierStage.ts`** — Converts frontier edges into `FrontierGraph` (edge map, adjacency, shared edges)
- **`regionStage.ts`** — Graham scan convex hull over owned star positions + frontier split nodes → raw convex hull in world coordinates. **`expandPolygon` removed** (user correctly identified it as a rendering concern)
- **`TerritoryEngineController.ts`** — Orchestrates stages, passes `CanonicalTerritoryState` downstream
- **`TerritoryRenderer.ts`** / `OwnerFillLayerRenderer.ts` / `BorderLayerRenderer.ts` — Render fills and borders from canonical state

Key fixes:
- Fixed `g.visible = true` in both layer renderers — GameCanvas blanket-hides all children each frame
- Removed `expandPolygon` from `regionStage` — pixel expansion is a renderer concern only
- No PIXI imports in compiler, no module-level mutable state

Key files modified:
- `pax-fluxia/src/lib/territory/compiler/regionStage.ts`
- `pax-fluxia/src/lib/territory/renderers/OwnerFillLayerRenderer.ts`

### 2. Builtin Theme System — Key Bug Fixed

**Root cause:** `getBuiltinGameThemes()` routed JSON values through `splitIntoCategories()` → `CATEGORY_KEYS`. Any key NOT explicitly listed in `CATEGORY_KEYS` was silently dropped — killing all `TERRITORY_ENGINE_*`, `DF_*`, `GRAPH_*`, `LANE_*`, `BORDER_*`, `STAR_LABEL_*`, etc.

**Fix:** `getBuiltinGameThemes()` now reads raw flat `values` dict directly from JSON, identical to how user-saved `GameTheme` objects work. No lossy category round-trip.

Key file: `pax-fluxia/src/lib/config/builtinThemes.ts` — `getBuiltinGameThemes()` function.

### 4. Audio Theme Sync Bug Fixed

**Root cause:** `AudioManager` has its own `$state` reactive mirrors (`masterVolume`, `soundVolumes`, `soundFiles`, `soundOffsets`) separate from `GAME_CONFIG`. When `themeStore.applyTheme()` wrote values to `GAME_CONFIG`, `AudioManager` state was never notified.

**Fix:** `themeStore.applyTheme()` and `themeStore.importTheme()` both call `audioManager.syncFromConfig()` immediately after applying theme values.

Key files: `pax-fluxia/src/lib/stores/themeStore.svelte.ts` (import + two call sites)
