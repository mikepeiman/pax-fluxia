# Session - 2026-05-28

## Grid Gradient Transition Trace Coupling

User-reported state:

- Enabling `Grid Gradient transition trace` also toggled the broad Render logs.
- Logs continued scrolling endlessly while the game was paused.

Root cause:

- `ControlsSection-Logging.svelte` explicitly set `(logFlags as any).renderer = true` when `GRID_GRADIENT_DEBUG_TRANSITIONS` was enabled.
- `transitionTraceLogger.ts` emitted through `log.renderer(...)`.
- That made a narrow Grid Gradient diagnostic depend on and mutate the broad renderer log channel.

Implementation:

- Added `log.gridGradientTrace(...)` to `logger.ts`.
- Changed `transitionTraceLogger.ts` to call `log.gridGradientTrace(...)`.
- Removed the automatic Render-log toggle from `ControlsSection-Logging.svelte`.
- Updated the config comment and settings search metadata to state that the Grid Gradient trace does not enable broad Render logs.
- Added an `AGENT.md` rule: narrow diagnostic toggles must not mutate broad log-channel toggles.

Validation:

- `bun test src/lib/territory/transitions/renderFamilyPreviousFrame.test.ts src/lib/territory/families/gridGradient/GridGradientFamily.test.ts src/lib/territory/families/gridGradient/gridGradientShaderFieldPacking.test.ts`
- `bun run build` in `pax-fluxia/`

## Grid Gradient Transition Trace Volume

User-reported state:

- The trace no longer logs while paused.
- While the game is running, the trace still emits many routine records for one transition.
- The message has a duplicated `[GG_TRANSITION]` prefix.

Root cause:

- `transitionTraceLogger.ts` deduped per stage, but it included progress buckets in every stage signature.
- Routine stages such as `family.update.entry`, `family.update.plan_and_progress`, `shader.update.entry`, `family.update.role_counts`, and `case.grid_gradient.after_update` therefore repeated as progress advanced.
- The helper passed `[GG_TRANSITION] ${label}` into `log.gridGradientTrace(...)`, while the logger already prepended `[GG_TRANSITION]`.

Implementation:

- Changed the trace helper to classify labels as `summary`, `structural`, or suppressed.
- Suppressed routine per-frame stages.
- Kept lifecycle/gate/rebuild/error records, deduped without progress churn.
- Kept `family.update.exit` as the only coarse progress summary.
- Removed the duplicated trace prefix.
- Added `transitionTraceLogger.test.ts` for the trace volume behavior.
