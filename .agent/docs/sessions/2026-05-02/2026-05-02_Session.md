# Session Notes - 2026-05-02

## Goal

Implement the agreed fix plan for the persistent Phase Edges fill/border divergence:

- make `Centered-blended borders` a border-only change
- preserve one canonical fill path
- make boundary fill ownership come from the ownership frontier rather than transition-only scene roles

## Findings

- The old blended branch was still special-casing fill geometry instead of only changing border presentation.
- `Inward Offset` and `Flush Boundary Fill` were effectively keyed off `scene` roles (`native` vs non-`native`), which mostly describes conquest-transition semantics, not the steady ownership frontier.
- The current `Centered-blended borders = Off` path was the best working reference because the user confirmed its fill behavior as the acceptable baseline.
- After that ownership-only fix, the remaining bug was still visible because centered-blended border geometry was being drawn on the abstract grid instead of the visible fill edge.
- The current live settings revealed a deeper contract bug: the active mode was `TERRITORY_FRONTIER_TECHNIQUE = marching_triangles_gradient`, and `METABALL_GRID_BORDER_BLEND` was still changing fill by pushing contour techniques onto scene-pair phase layers.
