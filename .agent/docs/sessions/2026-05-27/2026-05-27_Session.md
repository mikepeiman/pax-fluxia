User-reported state:

- Grid Gradient diagnostics showed transition timing and progress, but the fill rendered as snap pre/post with no visible transition.
- User specifically directed the investigation away from places where transition code merely appeared to exist and toward the connection point where it was not actually being rendered.

- Active worktree is `C:\Users\mikep\.codex\worktrees\9f22\pax-fluxia`.
- Current live worktree servers observed earlier in the session were on `localhost:1401` and `127.0.0.1:1464`; no listener was found on the earlier screenshot port `1441`.
- The stale `common/resources/settings-live/current-settings.json` was not treated as reliable live browser state.
- The Grid Gradient family already had code for shader-field progress and active transition cell roles, so the likely failure was upstream connection rather than missing draw logic.

Correction after user verification:

- The prior "pending preview before post-conquest ownership" explanation was not proven by live behavior. The user reported no visible change after that patch, so it must not be documented as the root cause.
- The only concrete implementation error from that pass was that Grid Gradient became the only render-family case to gate `activeRenderFamilyTransition` through a post-owner check before building its family input. That gate could erase transition input and has been removed.
- Current working hypothesis is still open: the transition may be disconnected at lifecycle/input, previous-frame selection, plan/classification, shader texture packing, shader uniform update, or visual shader consumption. The new diagnostics log all of those stages explicitly.

Implementation:

- Removed the Grid Gradient-specific post-owner gate and the now-unused `transitionHasPostConquestOwners(...)` helper.
- Kept `ownershipSnapshotHasPreviousConquestOwners(...)` so the previous-frame cache is used only when it still represents each conquest event's `previousOwner`; otherwise previous geometry is rebuilt from reverted stars.
- In `GameCanvas.svelte`, the previous-frame cache is used only if its ownership snapshot still has each event's `previousOwner`; otherwise previous geometry is rebuilt from reverted stars.
- Added gated Grid Gradient transition diagnostics under `GRID_GRADIENT_DEBUG_TRANSITIONS`. Logs use `log.renderer(...)` with the stable filter prefix `[GG_TRANSITION]`.
- Diagnostic stages now include transition lifecycle, territory scheduler, presentation queue, previous-frame cache/rebuild, Grid Gradient case input, plan/cache, role counts, shader texture packing, mesh setup, and shader uniform values.
- The Diagnostics panel also exposes active/drawable/mixing/offset cell counts, shader-side active counts, transition event/session counts, age/duration, and shader uniform progress.
