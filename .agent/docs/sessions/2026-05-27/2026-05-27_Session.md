## Grid Gradient Fill Transition Disconnection

User-reported state:

- Grid Gradient diagnostics showed transition timing and progress, but the fill rendered as snap pre/post with no visible transition.
- User specifically directed the investigation away from places where transition code merely appeared to exist and toward the connection point where it was not actually being rendered.

Investigation summary:

- Active worktree is `C:\Users\mikep\.codex\worktrees\9f22\pax-fluxia`.
- Current live worktree servers observed earlier in the session were on `localhost:1401` and `127.0.0.1:1464`; no listener was found on the earlier screenshot port `1441`.
- The stale `common/resources/settings-live/current-settings.json` was not treated as reliable live browser state.
- The Grid Gradient family already had code for shader-field progress and active transition cell roles, so the likely failure was upstream connection rather than missing draw logic.

Cause identified:

- `buildRenderFamilyTransitionLifecycle()` can expose a pending conquest preview as `activeTransition` before the active star ownership and geometry have advanced to the post-conquest state.
- Grid Gradient consumed that pending transition directly. That allowed it to build or keep a transition plan with a live clock but without a real PREV/NEXT geometry delta.
- `getTransitionDiagnosticPrevFrame()` was also used as the previous-frame source for render-family transitions and trusted `renderFamilyStableOwnership` without validating that it still represented the previous owner for every conquest event.

Implementation:

- Added `transitionHasPostConquestOwners(...)` and `ownershipSnapshotHasPreviousConquestOwners(...)` in `pax-fluxia/src/lib/territory/transitions/renderFamilyPreviousFrame.ts`.
- In `GameCanvas.svelte`, Grid Gradient now passes `activeTransition` into the family only after every conquest star in the transition has the event's `newOwner`.
- In `GameCanvas.svelte`, the previous-frame cache is used only if its ownership snapshot still has each event's `previousOwner`; otherwise previous geometry is rebuilt from reverted stars.
- Transition sessions are withheld from Grid Gradient while the active event is only a pending preview.
