`pax-fluxia/src/lib/territory/` - `pax-fluxia/src/lib/fx/` - `pax-fluxia/src/lib/animations/` - `pax-fluxia/src/lib/config/game.config.ts` - current rendering and animation specifications in `.agent/SPECIFICATIONS/`  ## Status Legend  - `Shipping`: in active use and visibly functional in the main runtime - `Integrated Experimental`: wired into the runtime or UI, but not yet trustworthy as long-term core - `Partial Attempt`: meaningful code exists, but architecture, quality, or completion is insufficient - `Documented Only`: described in docs or naming, not implemented as a first-class runtime behavior - `Historical/Superseded`: still present for reference or compatibility, but not suitable as future truth  ## Canonical Audit Taxonomy  - TerritoryGeometryMode - Territory is the most architecturally important and the least fully resolved.    - It has the most algorithms, the most naming drift, and the strongest need for a single canonical truth pipeline.  2. Stars and lanes are visually coherent and relatively stable.    - They mostly need better semantic mode naming and better integration into the final vocabulary.  3. Ships are not one mode family.    - They are a lifecycle graph with explicit sub-concerns: orbit, departure, travel, arrival, attack, conquest, damage, repair, and destruction.  4. Repair and destruction visuals are currently the clearest missing pieces.    - The gameplay model exists, but the rendering language does not yet expose those states with dedicated FX modules.
