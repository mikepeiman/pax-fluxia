### 1. B-42 Root Cause Traced
Traced active code path: DY4 → `legacy_pvv2` → PVV2 (PowerVoronoiRenderer).
Root cause: fills used `poly().fill()` (straight lines between vertices) while borders used `quadraticCurveTo()` (Bézier curves through midpoints). At concave corners, straight fill edges jut beyond the curved borders.
Key files: `PowerVoronoiRenderer.ts` (L859-885 → L859-934 after fix)

### 2. B-42 Fix
Rewrote `drawTerritoryFillWithHoles()` to construct fill paths using same `quadraticCurveTo` through midpoints as `drawBorderPolylines()`. Both fills and holes use the closed-polygon Bézier: start at midpoint(last,first), curve through each vertex to the next midpoint, wrap back.
Key files: `PowerVoronoiRenderer.ts`

### 3. V3.2 Four-Concern Architecture Plan
User insight: DY4 is a border animation, not a renderer — PVV2/PVV3 should be style modes. Catalogued all legacy modes into four concerns:
1. Data Generation Engine (Power Voronoi, FG2, DF, Metaball, Pixel, Contour)
2. Visual Style (Smooth Polygon, Shell Contour, Distance Gradient, etc.)
3. Border Transition (DY4, Smooth Morph, Segment Lerp, None)
4. Fill Transition (Crossfade, None)
