### 7. Phase 1 Architecture Refactor — New Territory Layer
Created `src/lib/territory/` with 12 new files per Plan V3 architecture:

**Compiler layer** (`territory/compiler/`):
- `types.ts` — all canonical interfaces (MetricState, FrontierGraph, TerritoryRegion, CanonicalTerritoryState, TransitionPlan, CompileError)
- `metricStage.ts` — multi-source top-2 Dijkstra with MSR seed offset, MinHeap PQ, O(E log V)
- `frontierStage.ts` — analytical lane split, builds FrontierGraph with each owner-pair once; CX deleted
- `regionStage.ts` — closed owned loops with componentId tracking (DX replacement as post-process)
- `frontierFitter.ts` — RDP simplification (straight), angle quantization (segmented), curved stub
- `TerritoryTransitionPlanner.ts` — pairId-matched edge correspondence planning; no clock state
- `TerritoryCompiler.ts` — orchestrator, typed error propagation, never emits fallback geometry

**Engine layer** (`territory/engine/`):
- `TerritoryEngineController.ts` — class-encapsulated, fingerprint dirty detection, compiler invocation

**Render layer** (`territory/render/`):
- `buildFillMeshCache.ts` — TerritoryRegion[] → FillMeshCache (reads regions, no ownership logic)
- `buildBorderMeshCache.ts` — FittedFrontier[] → BorderMeshCache (reads frontier, no ownership logic)
- `OwnerFillLayerRenderer.ts` — class-encapsulated fill drawing, hole/enclave cut support
- `BorderLayerRenderer.ts` — class-encapsulated border drawing, 50/50 owner color blend
- `TerritoryRenderer.ts` — unified entry, steady-state and transition pass; both calls share identical CanonicalTerritoryState
