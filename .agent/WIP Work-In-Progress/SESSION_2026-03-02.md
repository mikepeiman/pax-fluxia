## Evening Session: Territory Rendering

### 🟡 F-135: Junction Angle Equalization (Contour renderer) — Shelved
- **Goal**: Equalize angles where 3+ territory owners meet
- **v1**: Snap-key vertex matching — failed because marching squares edge midpoints are pairwise (between 2 owners), so ≥3 match never fires
- **v2**: Grid-based junction detection (scan cell corners for ≥3 owners) — correctly detects junctions, pulls acute vertex back via bisector
- **v3**: Multi-vertex collapse — finds ALL nearby vertices per polygon within searchRadius, collapses to new position
- **Outcome**: Still insufficient. Marching squares geometry is fundamentally too noisy as a starting point. User observed: "it pulls the point back but fails to redraw connecting borders"
- **Decision (D-34)**: Abandon contour/marching squares approach. Use Voronoi as the foundation instead.

### 🟢 F-138: Voronoi Merged Territories — Planned
- **Approach**: Use d3-delaunay (already installed) Voronoi cells, merge same-owner adjacent cells into unified polygons
- **Key insight**: Voronoi gives mathematically clean geometry, natural ~120° junction angles, proper topology
- **Boilerplate**: Created `MergedVoronoiRenderer.ts` (duplicate of `VoronoiRenderer.ts` with TODO markers)
- **Pipeline**: d3-delaunay cells → merge same-owner → min star margin → Bézier arc junctions → Chaikin smoothing

### 🟢 F-139: Minimum Star Boundary Margin — Planned
- Territory boundaries must be ≥5 orbit radii from any star center
- Prevents orbit ring clipping through boundary edges

### 🟢 F-137: Periphery Coverage — Documented
- Same-owner stars at map edge should have territory begin on the *inside* of the connecting lane and fully cover the outside
