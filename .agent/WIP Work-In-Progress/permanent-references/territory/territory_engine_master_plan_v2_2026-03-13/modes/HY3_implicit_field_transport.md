# HY3 Implicit Field + Transport

## Plain-Language Explanation

`HY3` combines implicit field-based frontier truth from `FG3` with transport-style dynamic motion from `DY4`. It is the smoothness-oriented hybrid among the current five plans.

## Current Implementation State

- Status: partial
- Native maturity: not yet native end-to-end
- Current role: selectable comparison hybrid plan for smooth field-plus-motion experimentation

## Current Live Routing And Backend Reality

- Live only when `hybrid` mode is active and `HY3` is selected
- Current backend reality: adapter-backed through `DF`
- Current static leg: `FG3`
- Current dynamic leg: `DY4`

## Target Shape

`HY3` should become the native hybrid plan for smooth implicit-contour motion with explicit cost and correction boundaries.

## Backend Fit

- `DF`: strongest current conceptual fit because both legs are field or transport adjacent
- `PVV3`: important parity target once the contract is native
- `PVV2`: maintained comparison target

## Dependencies

- `HY-000` common hybrid route contract
- `FG3` native implicit contour publisher
- `DY4` native transport/correction path

## Atomic Tasks

## Acceptance Checks

- Hybrid motion remains smooth without masking route truth errors
- Cost and correction tradeoffs are documented and demoable
- The field-based static leg remains identifiable in trace output
