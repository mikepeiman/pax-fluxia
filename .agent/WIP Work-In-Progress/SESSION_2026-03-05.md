## Technical Notes

### Shader Architecture (Current State)
```
CPU side:                                     GPU side:
stars[] + Dijkstra distances                  Fragment shader loop (0..uNumStars):
  → buildStarDataTexture()                      → decode star pos, owner, dijkstra from texture
  → 4-row RGBA texture (pos, dist, owner, prev) → influence = pixelDist + w * dijkstra
                                                → track best + enemy + second-best
                                                → border: fwidth-normalized, 50/50 color blend
                                                → junction smoothing: alpha *= smoothstep(0, uSmoothing, enemyGap)
                                                → edge fade: symmetric using uContentMinX/Y
```

### Key Uniform Registry
| Uniform | Type | Default | Purpose |
|---------|------|---------|---------|
| uSmoothing | f32 | 30 | Junction corner rounding radius |
| uContentMinX/Y | f32 | computed | Star bounding box min for symmetric edge fade |
| uInfluenceWeight | f32 | 1.0 | Graph distance multiplier (0=pure Voronoi) |
| uBorderWidth/Softness | f32 | 15/10 | Border band (fwidth-normalized) |

### Lessons Learned (2026-03-04)
- **Edge fade asymmetry**: `GAME_WIDTH = maxX + pad` is NOT symmetric if stars don't start at (0,0). Must compute actual content min bounds.
- **Border width**: `abs(influence₁ - influence₂)` doesn't map to constant screen distance. Use `fwidth()` to normalize.
- **Junction smoothing**: Must use `enemyInfluence` (different owner) not `secondInfluence` (any owner) — otherwise creates gaps within same-owner territory.
- **Mesh caching**: Must track ALL values that affect geometry (dimensions AND expansion) for proper invalidation.
- **Zoom anchor**: `handleWheel` and `applyZoomTransform` MUST use the same reference frame for coordinate centering.
