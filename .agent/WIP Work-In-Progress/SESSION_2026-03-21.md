### 2. Split Easing + Island Dissolution
Split the single easing curve into `tGhost` (easeOutCubic) for ghost movement and `tConquest` (easeInCubic) for conquered star weight ramp. Added island dissolution animation — when a single-star territory is conquered, its polygon shrinks to center with alpha fade using PIXI v8 API.

Key files: `PowerVoronoiRenderer.ts`

### 3. VS Transition Mode Selector (6 Modes)
Added `VS_TRANSITION_MODE` config key with 6 switchable modes: `dual_ghost`, `no_loser` (default), `no_ghosts`, `matched_ease`, `sequential`, `linear`. Each mode has different easing and ghost strategies implemented via a `switch` statement in the per-frame ghost loop. Added `<select>` dropdown in the Conquest panel under "VS Transition" heading.

Key files: `PowerVoronoiRenderer.ts`, `game.config.ts`, `settingsDefs.ts`, `ControlsSection-Conquest.svelte`

### 4. Animation Timing Slider Fix
The "Conquest Animation Timing" slider (`TERRITORY_TRANSITION_MS`) wasn't affecting VS transitions because:
1. `VS_BIND_TO_TICK` overrode the duration with `tickMs` (fixed: now only caps, doesn't override)
2. `VS_VICTOR_TRAVEL_MS=80` and `VS_LOSER_TRAVEL_MS=80` overrode `baseDuration` via `80 || baseDuration = 80` (fixed: now uses `baseDuration` directly)
3. Outer `rawT` used stale `smoothTransitionStart` instead of weight-lerp timing (fixed: conditional on `s.weightLerpActive`)

Moved the slider from Territory panel to Conquest panel. User confirmed slider works after fix.

Key files: `PowerVoronoiRenderer.ts`, `ControlsSection-Conquest.svelte`, `ControlsSection-Territory.svelte`
