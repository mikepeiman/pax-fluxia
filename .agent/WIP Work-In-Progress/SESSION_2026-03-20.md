### 3. Ghost-Site Weight Interpolation Transitions
Implemented a new transition approach for territory conquests:

**Problem:** Old weight-lerp (just adjusting site weights) barely moved boundaries because ownership drives boundary position, not weight (~45px).

**Solution — Ghost Sites:**
- For each conquered star, create a **ghost PowerSite** at the same position with the OLD owner
- Ghost weight fades W→0 while new owner weight grows 0→W
- Two competing sites at the same spot create smooth boundary handoff
- Modified `computeGeometry0319` to accept `extraSites` parameter for ghost injection
- Added `changedSitePrevOwners`, `weightLerpGhostSites`, `weightLerpGhostWeightStart` to `PVV2RendererState`

**Bugs fixed:**
- White borders during transition: `polyline.color=0` → `0 || 0xffffff` gave white. Fix: compute from `ownerPairKey` via `colorUtils.getPlayerColor()` + `blendColors()`
- Fill disappearance after conquest: `smooth-anim` block cleared `fillGraphics` every frame, NONE branch drew nothing. Fix: static-fallback draws fills + borders from `lastMergedTerritories`
- Stale transition flags: cancelled `isSmoothTransitioning`/`isFillTransitioning`/etc. on both weight-lerp setup and completion

**Key files modified:**
- `PowerVoronoiRenderer.ts` — ghost site state, weight-lerp draw loop, static-fallback in NONE branch
- `Geometry_0319.ts` — `computeGeometry0319` accepts optional `extraSites: PowerSite[]`
