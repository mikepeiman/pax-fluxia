### 1. B-42: Fill/Border Divergence (RESOLVED)

**Root cause:** `PowerVoronoiRenderer.ts` was calling `chaikinSmoothPolygon` independently on fill polygons at render time. This introduced timing divergence — fills were smoothed separately from borders, causing jutting corners.

**Fix:**
- Removed render-time fill smoothing from `drawTerritoryFillOnly`
- Applied `chaikinSmoothPolygon` to `mergedTerritories` and enclave holes in `pvv2MetricStage.ts` at geometry stage
- Both fills and borders now smoothed with same `chaikinPasses` from one source

**Key files:** `pvv2MetricStage.ts` (Stage 6), `PowerVoronoiRenderer.ts` (`drawTerritoryFillOnly`)

---

### 2. VORONOI_BORDER_SMOOTH Slider Fix (RESOLVED)

**Root cause:** `VORONOI_BORDER_SMOOTH` was in `buildVisualFingerprint` → changing it only triggered a visual redraw, not a geometry rebuild.

**Fix:** Moved to `buildShapeFingerprint` → slider now triggers full geometry rebuild with correct `chaikinPasses`.

---

### 3. Boundary-Pinned Chaikin (RESOLVED)

**Problem:** Chaikin smoothing cuts polygon corners at world edges → fills pull away from world boundary leaving dark gaps.

**Fix:** Modified `chaikinSmoothPolygon` to accept `worldW`, `worldH`, `pad` parameters. At each pass, vertices within `eps=6` of the padded clip boundary are preserved (not cut). Interior corners still smooth normally.

**Key file:** `pvv2MetricStage.ts` — `chaikinSmoothPolygon` function

---

### 4. Junction-Pinned Chaikin (RESOLVED)

**Problem:** At 3-way territory junctions, Chaikin cuts each polygon's corner → three fills all pull away from junction vertex, creating triangular gap.

**Fix:**
- Added `extractJunctionVertices(cells)` — builds Set of ptKey strings for vertices shared by 3+ cells
- Extended `chaikinSmoothPolygon` to accept `pinnedPtKeys?: Set<string>` — adds junction pinning on top of boundary pinning
- Junction set computed early (after cell build, before merge) and passed to all smoothing calls

**Key file:** `pvv2MetricStage.ts` — `extractJunctionVertices`, `chaikinSmoothPolygon`

---

### 5. Transition Easing Expansion (RESOLVED)

**Problem:** Only `cubic`, `back` (overshoot), `elastic` (bouncy) were available. No way to get clean deceleration without backward travel.

**Fix:** Added 4 new easing functions to `borderTransition.ts`:
- `easeOutCubic` — decelerates, no overshoot
- `easeOutQuad` — lighter deceleration
- `easeLinear` — constant speed
- `easeInOutSine` — gentle S-curve

UI dropdown in Territory settings now shows all 7 options. Both `GraphicsPathMorpher` and `RopeBorderRenderer` constructors updated.

**Approach implemented:**

- `lastWorldBorderPolylines` module cache for transition-end redraw
