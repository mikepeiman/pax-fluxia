### 1. Category Theme System
Built 3-tier architecture: `categoryThemes.ts` maps each panel section to its config keys, `CategoryThemeBar.svelte` provides per-section save/load/apply UI. Drawer with chips, side-by-side buttons, file download on save. Wired to all 12 ControlsSection panels. Fixed .toFixed() null crashes, syncFromConfig destructuring, and preset list reactivity.

### 2. Pixel Territory Renderer
New 3rd territory mode — nearest-neighbor fill with spatial grid optimization. Added independent PIXEL_* controls (resolution, blend power, alpha, saturation), HSLA color controls. Pattern fills: stripes, crosshatch, dots with per-player golden-angle rotation for visual differentiation.

### 3. Pixel Territory Performance
- Throttle recompute to max 2Hz (500ms)
- Adaptive render throttle: 5× last duration prevents blocking
- Hierarchical adaptive resolution: 8×8 tile pass, fine only at ~10-15% boundary tiles
- **Web Worker offload**: full computation off main thread for zero-jank

### 4. Territory Corridors & Borders
- Power-weighted blending for rounded borders (PIXEL_BLEND_POWER)
- Per-faction corridor boost for connected same-owner stars
- Rewritten from scratch as per-faction influence accumulation
- Border detection uses ownership grid (not RGB comparison)
- Territory edge fade: smooth alpha falloff at gameboard edges (0-500px configurable)

### 5. Orphaned Slider Reactivity (6 sliders)

Fixed: `SHIP_VISUAL_RADIUS`, `STAR_GLOW_RADIUS_MULT`, `STAR_GLOW_INTENSITY`, `LANE_OFFSET_PX`, `LANE_CONVERGENCE`, `LANE_CONVERGENCE_POINT`.

### 6. Capsule Corridor Bridges
**Problem:** Connected same-owner stars had disconnected territory when enemy stars were closer to the link midpoint.

**Solution:** For each game connection where both endpoints share an owner, create a capsule (line segment + radius). Pixels inside any capsule are force-assigned to that owner before Voronoi distance checks.

- `pixelTerritory.worker.ts` — `segDistSq()`, `corridorOwner()`, early-exit
- `PixelTerritoryRenderer.ts` — builds `corridorSegs[]` from connections
- `GameCanvas.svelte` — passes `connections` to renderer

### 7. Master Theme System

- **Import/Export:** File picker validates `GameTheme` shape, JSON download on save
- MP lobby: `updateConfig/updateRoomOptions` handlers for theme sync

### 8. CIELAB ΔE Perceptual Color Spacing
Replaced even-hue (360°/N) with CIELAB-aware greedy algorithm:
1. 72 candidate hues at 5° intervals (skip ±40° near human blue)
2. Convert to Lab via sRGB → XYZ → Lab (D65)
3. Greedily pick max min-ΔE to all chosen colors

### 9. Background Image Controls
Live background picker in Map & Grid panel: thumbnail grid with "None" option, auto-discovers from `/api/backgrounds`, persists to localStorage, dispatches `pax-bg-change` custom event for live PIXI texture swap.

### Key Design Decisions
- **Capsule vs graph-bridge:** Geometric capsule over BFS — simpler, O(segments × pixels)
- **CIELAB over CIEDE2000:** CIE76 sufficient for 4-8 players
- **Hierarchical resolution:** Coarse tile pass eliminates 85-90% of pixels from fine computation
- **Worker over SharedArrayBuffer:** Simpler postMessage approach, sufficient for 2Hz recompute
