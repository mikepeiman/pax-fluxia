2. **Transition Geometry Shatter (DY4 / OT)**
   - **Fix**: Rewrote `alignPolygon()` to use strict `N` array length instead of `N-1`.
   - **Why**: `resamplePolygon()` does not output end-duplicated arrays. `alignPolygon()` was blindly dropping the final vertex (index 63), causing massive tears and criss-cross geometry in transition frames.
   - **Files**: `OptimalTransportBorderTransition.ts`

3. **Transition Fill Lag (PixiFillPresenter / Canonical-Clean)**
   - **Fix**: Injected `graphics.beginPath()` before poly/fill commands.
   - **Why**: PixiJS v8 inherently accumulates polygons unless explicitly separated. The transition generated 50+ polygons/frame; without `beginPath`, Pixi aggregated them into one massive self-intersecting shape, causing a 300ms+ earcut lag spike and visual shattering.
   - **Files**: `PixiFillPresenter.ts`, `PixiBorderPresenter.ts`

### 4. Unified Fill+Stroke Transition Architecture
- **Fix**: Replaced the separate border transition calculation pipeline with a `graphics.stroke()` call directly inside the fill presenter. Guted `activeBorderPlan`, `OptimalTransportBorderTransition`, and all related UI configurations from the layer coordinator and dev tools.
- **Why**: The independent border transition (running on the CPU) was lagging frames and visually diverging from the fill regions (producing shattered/criss-crossed borders that drifted from their owner regions). Bringing them into a single-pass stroke+fill ensures perfect alignment automatically, dropping the compute overhead entirely.
- **Files**: `PixiFillPresenter.ts`, `TransitionLayerCoordinator.ts`, `TerritoryRuntimeCoordinator.ts`, `TerritoryRuntimeState.ts`, `TransitionSnapshotRecorder.ts`, `ControlsSection-Territory.svelte`, `FillDrawCommandBuilder.ts`
