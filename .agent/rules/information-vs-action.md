# Information vs. Action

## MANDATORY: Distinguish questions from directives before acting.

### Questions = Report Only
If the user asks a question — "what do I need to do?", "is there anything to update?", "what's wrong?" — **report findings only.** Do NOT make changes.

### Directives = Proceed
If the user gives a directive — "update X", "fix Y", "rename Z" — proceed with changes.

### When in Doubt
Ask: "Would you like me to go ahead and make these changes, or just report what needs doing?"

### Infrastructure is Always Explicit
Git remotes, build configs, CI pipelines, deployment settings — **ALWAYS require explicit instruction.** Never infer. Never change preemptively.
